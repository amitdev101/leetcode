This file is mandatory and must act as a book index + dashboard.

It must include:
- title and overview
- "Start Here" section
- reading paths (beginner / contributor / architecture)
- chapters → topics → subtopics hierarchy
- module directory
- feature coverage summary
- recent updates
- remaining gaps
- difficulty levels
- estimated reading time
- links to ALL docs

Related topics

Each module must include:

Code Example Rule

Every example must include:

what it demonstrates
why it matters
how to modify it
expected behavior/output

Avoid trivial examples.

Quality Gate (STRICT)

A feature can only be marked covered if:

explanation exists
beginner can understand it
at least one real example exists
module/file mapping exists
testing/debugging notes exist OR are explicitly missing

Otherwise:
→ do NOT mark as covered

Write to:
