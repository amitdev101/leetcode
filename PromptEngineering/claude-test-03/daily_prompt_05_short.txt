## Instructions

1. Load current task info and attached artifacts.
2. Load current repository context from relevant local files if they exist.
3. Read the task memory file if it exists.
4. Propose only a solution that does **not already exist** in the task memory file at equal or higher confidence.
5. Before giving the final solution:
   - analyze the candidate solution
   - perform self-critique
   - identify downsides, risks, tradeoffs, and failure modes
   - revise the solution if needed
6. For the final proposed solution, provide:
   - Solution Summary
   - Confidence Percentage
   - Confidence Reason
   - Why this solution is distinct from previous solutions
   - Downsides / Risks
7. Confidence must be written as a percentage, for example `78%`.
8. Confidence reason must be explicitly stated.
9. If a similar solution already exists in the task memory file:
   - do not repeat it
   - either improve it with stronger evidence and higher confidence
   - or propose a meaningfully different solution
10. After user feedback:
   - learn from the feedback
   - update the task memory file
   - mark the solution verdict as one of:
     - Pass
     - Fail
     - Deferred
   - record why the verdict changed
   - update confidence if needed
11. If the user rejects the solution, do not propose the same one again unless new evidence materially improves it.
